# 灵感记录技能

## 核心理念

- **不假大空**：像两个人聊天，用真实有内容的语言
- **不堆书面语**：避免"赋能"、"抓手"、"闭环"等空洞词汇
- **平和务实**：坦诚表达，知道什么就是知道，不知道就说还在探索
- **段落写作**：周报像写文章，分模块但不堆列表，用自然段落表达思考

## 风格定义

### 我的风格（平和务实）

```
- 语气：像给一个懂行的朋友写信，自然、有温度
- 结构：分段落的文章，不是列表和提纲
- 反思：可以写"还在探索"、"不确定"、"需要验证"
- 行动：写真实的下一步想法，不用硬凑
```

### 老板的风格（务实直接）

```
- 语气：像写商务备忘录，直接、有信息量
- 结构：分段落的文章，每个模块有独立的自然段落
- 结论：有明确的判断和立场，不说正确的废话
- 行动：每条指向可执行的下一步，不用罗列很多
```

## 工作流程

### 阶段 1：接收与解析

1. 读取 `skill.log.json`，获取两个汇总文档 token
2. 判断灵感来源（我的 / 老板的）
3. 解析内容，提取标题、标签、类型

### 阶段 2：调研（两者共用）

4. 智能关联召回 — 搜索 `memory/inbox/` 历史记录
5. 拆分关键问题 — 提取 2~5 个关键问题
6. 联网搜索 — 对每个问题 `web_search`，最多 3 并发（注意速率限制）
7. 生成调研报告 — 整合搜索结果，按"关键问题 → 发现 → 来源"结构写作；每个来源条目须包含：来源、摘要、与灵感的关联（支持/质疑/印证/补充）
8. 合并到汇总文档 — 将调研报告核心内容追加进汇总文档（不是只加链接）
9. 更新本地文件 — 本地 `md` 同时更新
10. 更新 `skill.log.json` — 记录 `report_doc` token

### 阶段 3：输出（分来源）

**我的灵感 → 我的汇总文档：**
- 写入本地 `memory/inbox/💡_YYYY-MM-DD-HHMM.md`
- 追加到我的灵感汇总文档
- 更新 `skill.log.json`

**老板的灵感 → 老板汇总文档：**
- 写入本地 `memory/inbox/📋_YYYY-MM-DD-HHMM.md`
- 追加到老板灵感汇总文档
- 更新 `skill.log.json`

## 周报生成工作流

### 触发词

- 「我的周报」/ 「生成我的周报」 → 生成我的风格周报
- 「老板周报」/ 「生成周报给老板」 → 生成老板风格周报
- 可指定周期：「上周周报」、「这周的周报」、「X月周报」

### 步骤 1：确定周期

| 用户说 | 周期 |
|--------|------|
| 「我的周报」/ 「老板周报」（无指定） | 本周（本周一至今） |
| 「上周周报」 | 上周一至上周日 |
| 「X月周报」 | 指定月份全部灵感 |

### 步骤 2：收集灵感

1. 读取 `skill.log.json`
2. 根据 owner 筛选对应灵感列表（mine / boss）
3. 按时间过滤，只保留周期内的灵感
4. 对每条灵感读取本地 `.md` 文件或汇总文档内容
5. 读取关联的调研报告（`report_doc.token`），提取核心段落

### 步骤 3：内容提炼与改写

**改写原则（两者通用）：**
- 把调研报告的书面语改成日常表达
- 删除空洞词汇（"赋能"、"抓手"、"闭环"、"生态"、"赛道"等）
- 每句话都有具体信息，不说空话
- 宁可少写一条，也不凑字数
- 分段落写作，不用列表和提纲

### 步骤 4：输出

- 使用 `feishu_create_doc` 创建飞书文档
- 本地备份到 `memory/weekly/my/` 或 `memory/weekly/boss/`
- 更新 `skill.log.json` 中的周报记录
- 返回文档链接给用户

## 周报模板

### 我的周报模板

**定位**: 每周在跟自己对话，说说这周在琢磨什么。不是写给别人看的报告，是记录自己的思考过程。

**范式要点**:
- 开头方式固定：直接点题，如"关于AI PC的定义。"或"关于这周在琢磨的事。"
- 结构是自然段落：每段一个完整的意思，不堆列表，不做编号
- 思考有连续性：周报之间有关联，不是一周一周孤立写
- 用故事和类比来表达：不是用数据说服，是用具体的场景
- 带感情色彩：有判断、有困惑、有危机感，不假装客观
- 结尾停在问题本身：最后一个自然段落的最后一句是问句或判断，不做总结收尾，不加"再想想"之类的废话

**模板**:

```markdown
# X月第X周

关于[主题]。

（直接开始写第一段，固定用"关于XXX"开头。后面段落跟上去，每段一个完整的意思，段和段之间有关联）

（最后一段停在问题本身或判断本身，最后一句是问句或判断，不做总结收尾）
```

**内容组织原则**:
- 提取本周有感触的灵感内容，改写成自然段落
- 调研报告的内容要转成自己的话，不用书面语
- 保留真实感受：无力感、危机感、兴奋感都可以写
- 困惑和失误可以直接写，不用假装都有答案
- 每条灵感提炼1-2个核心想法，不贪多
- 段落之间有逻辑关联，但不显性说明

**禁用清单**:
- ❌ 废话开头："本周完成了几项工作"、"光阴似箭"、"这周主要"
- ❌ 列表和编号
- ❌ 空洞词汇：赋能、抓手、闭环、生态、赛道、沉淀、拉齐、对齐等
- ❌ 结尾总结收尾："综上所述"、"总的来说"、"再想想"
- ❌ 用数据说服（数字只在具体场景描述中出现）

### 老板周报模板

**定位**: "我向老板汇报"的口吻——说清楚我这周的想法、判断、下一步。不是老板在命令你，是你跟老板讲你在想什么、打算怎么做。

**范式要点**:
- 开头给结论，不铺垫，不用"最近在思考"这种废话
- 用自己的观察和调研做支撑，不是引用权威
- "我想"引出下一步，说具体要做什么，带出负责人和时间
- 结构是"判断 + 路径 + 具体行动"
- 不讲道理，直接说判断
- 不用"核心判断"、"关键发现"等标题
- 不用列表和提纲格式
- 周报正文不用具体数字（42%、120亿等）

**模板**:

```markdown
# X月第X周

开头直接给结论。

用自己的观察和思考来说明这个判断。

接下来顺着这个思路做几件具体的事。用"我想"开头，分段写具体要做什么，有范围和时间的直接写出来。

还有一件要交代的事。
```

**禁用清单**:

结构禁用：
- ❌ "核心判断"、"关键发现"、"下一步行动"等标题
- ❌ 列表和提纲格式
- ❌ "1、2、3"编号列表

词汇禁用：
- ❌ 空洞词汇：赋能、抓手、闭环、生态、赛道、沉淀、拉齐、对齐、打透、穿透、构建、建立、完善、优化、迭代、升级、共振、协同、联动、发力、布局、破局、引领、驱动、支撑、激活
- ❌ 斩钉截铁的行动指令：如"这周开始做"、"必须"、"务必"——让判断本身带出行动方向，不靠决心词

数据禁用：
- ❌ 正文引用具体数字（42%、120亿等）

---

## 禁用说明

- "只记录，不研究" → 跳过调研步骤
- "不用写周报" → 跳过周报生成
- "只记录" → 跳过调研和周报
- 可组合："只记录，不研究，不用写周报"
